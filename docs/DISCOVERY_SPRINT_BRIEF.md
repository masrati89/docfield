המשתמש (חיים) הבהיר שיש **3 וריאציות של דוח, אבל DB אחד**:

| וריאציה | report_type               | צ'קליסט | מחירים | תקן ישראלי | המלצה | תבנית PDF                 |
| ------- | ------------------------- | ------- | ------ | ---------- | ----- | ------------------------- |
| **A**   | `handover_with_checklist` | ✅      | ❌     | ❌         | קצרה  | Protocol (עם דף צ'קליסט)  |
| **B**   | `handover_no_checklist`   | ❌      | ❌     | ❌         | ✅    | Protocol (בלי דף צ'קליסט) |
| **C**   | `bedek_bayit`             | ❌      | ✅     | ✅         | ✅    | BedekBayit                |

### החלטות סגורות (לא לשאול)

1. **DB אחד לכל 3 הוריאציות** - שדות אופציונליים
2. **`report_type` enum** ב-`delivery_reports`
3. **MVP = רק וריאציה A**
4. **B + C ב-v1.1** - DB יתמוך בהם, אבל UI לא נבנה עכשיו
5. **2 PDF templates בלבד** - Protocol + BedekBayit. B = A מינוס דף הצ'קליסט.
6. **deploy יעד: 9 במאי 2026**

### Reference files (ב-`~/infield/docs/reference/`)

1. **`inField-Protocol-PDF.jsx`** - source of truth של וריאציה A. **חובה לקרוא.**
2. **`inField-BedekBayit-PDF.jsx`** - לעיון בלבד, לתכנון DB עתידי
